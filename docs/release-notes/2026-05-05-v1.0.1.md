# GasStation 1.0.1 릴리즈 노트

릴리즈 날짜: 2026-05-05

이 릴리즈는 남은 backlog risk를 줄이는 패치 릴리즈입니다. 제품 흐름의 큰 방향은 유지하면서 secret handling 안내, app chrome, settings persistence boundary, Gradle test convention, 검증 문서를 현재 구현과 맞췄습니다.

## 무엇이 바뀌었나

### 실행과 보안 경계

- `prod` 실행에는 사용자 로컬 `opinet.apikey`가 필요하다는 점을 README와 검증 문서에 명확히 남겼습니다.
- 버전 관리되는 프로젝트 루트 `gradle.properties`에 실제 key를 두지 않는 운영 기준을 문서화했습니다.
- Android backup/data extraction이 로컬 캐시, 가격 히스토리, watchlist, 설정을 내보내지 않도록 하는 보안 경계를 README에서 설명합니다.

### 앱 chrome

- `MainActivity`가 `GasStationThemeDefaults.statusBarStyle`을 사용해 status/navigation bar 색상과 icon contrast를 적용합니다.
- system bar 정책을 app 단위 테스트로 고정해 테마 회귀를 더 빨리 잡을 수 있게 했습니다.

### 설정 저장 경계

- `core:datastore`는 `StoredUserPreferences` storage DTO와 serializer만 소유합니다.
- `data:settings`가 storage DTO와 domain `UserPreferences` 사이의 mapper를 소유합니다.
- 저장된 enum name이 현재 앱에서 알 수 없는 값이면 domain default로 fallback합니다.

### Gradle convention 정리

- Android library 공통 unit test 의존성은 `gasstation.android.library` convention이 제공합니다.
- Compose Android library의 Compose UI test/debug 의존성은 `gasstation.android.library.compose` convention이 제공합니다.
- 각 모듈 build file에는 모듈별로 필요한 의존성만 남겨 변경 표면을 줄였습니다.

### 위치 Geocoder 검증

- API 33+ Geocoder callback path를 실제 기기나 에뮬레이터에서 확인하는 `AndroidAddressResolverDeviceTest`를 추가했습니다.
- Provider 출력은 환경별로 달라질 수 있어 주소 문자열 대신 terminal domain result 도달을 검증합니다.

### 문서와 검증

- README의 아키텍처 그림과 문서 지도를 현재 모듈 경계에 맞췄습니다.
- `docs/module-contracts.md`, `docs/architecture.md`, `docs/state-model.md`, `docs/test-strategy.md`, `docs/verification-matrix.md`, `docs/improvement-analysis.md`가 이번 구현 결과를 반영합니다.
- `docs/verification-matrix.md`에 위치 Geocoder 기기 smoke 명령을 추가했습니다.

## 아직 실행하지 않은 외부 단계

- Opinet provider key rotation은 provider credential과 사용자 승인이 필요해 이 릴리즈에서 실행하지 않았습니다.
- git history rewrite와 force push는 파괴적 git 작업이므로 별도 승인 없이 실행하지 않았습니다.
- 기기 기반 `connectedDebugAndroidTest`와 `connectedDemoDebugAndroidTest`는 연결된 Android device/emulator가 있을 때 실행합니다.
