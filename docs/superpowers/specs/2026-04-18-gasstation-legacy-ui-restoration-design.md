# GasStation Legacy UI Restoration Design

## 목표

리뉴얼 이후 흐려진 GasStation의 시각 정체성을, 원래 앱이 갖고 있던 `노랑 + 검정` 브랜드 톤과 높은 정보 밀도를 기준으로 복원한다. 복원 범위는 `주유소 목록`, `설정`, `관심 비교` 세 화면 전체이며, 신규 기능인 watchlist, stale 상태, 새로고침, 권한/GPS 안내는 예전 화면 문법 안에 자연스럽게 통합한다.

## 복원 원칙

### 1. 원형 우선

시각 스타일과 레이아웃은 현재 Material 3 기본 화면보다, 기존 앱의 인상과 사용 흐름을 우선한다.

- 고정 브랜드 팔레트 사용
- 검정 앱바, 노랑 배경, 흰 카드 내부 구조 복원
- 큰 숫자 중심 정보 위계 복원
- 설정 2단계 탐색 구조 복원

### 2. 기능은 현재 구조를 유지

도메인, 데이터, 내비게이션, 상태 모델은 현재 멀티모듈 구조를 유지한다. 이번 작업은 주로 `designsystem`과 `feature` 레벨의 UI/상호작용 복원에 집중한다.

### 3. 새 기능은 옛 문법에 맞춘다

watchlist, stale, 새로고침, 권한/GPS 상태는 새 화면 스타일을 만들지 않고 기존 브랜드 문법으로 흡수한다.

## 화면별 목표

### 주유소 목록

기존 `HomeScreen`의 인상을 Compose 멀티모듈 구조 안에서 재구성한다.

핵심 요소:

- 전체 바탕은 노랑 계열
- 상단바는 검정 배경 + 노랑 텍스트/아이콘
- 정렬 기준은 앱바 타이틀에서 직접 토글
- 설정 버튼은 우측 액션에 유지
- 관심 비교 진입도 같은 앱바 액션 문법 안에 배치
- 리스트 카드는 검정 외곽/흰 내부 구조를 사용
- 가격/거리 숫자는 화면에서 가장 강한 타이포 우선순위
- 브랜드/유종/보조 정보는 작은 회색/검정 보조층으로 배치

신규 상태 흡수 방식:

- `isStale`: 카드와 분리된 상단 정보 배너로 노출
- `isRefreshing`: 간단한 상태 배너 또는 상단 인라인 상태로 노출
- `isWatched`: 카드 우측 별 액션 유지, 기존 카드 문법을 해치지 않도록 단색 아이콘 사용
- 권한/GPS 비활성: 현재의 일반적인 빈 화면 대신, 예전 스타일과 맞는 큰 안내 카드로 표시

### 설정

현재 한 화면 안의 섹션형 카드 구성을 제거하고, 예전처럼 `설정 목록 -> 상세 선택` 2단계 구조로 복원한다.

구성:

- `SettingsRoute/Screen`은 항목 목록 화면 역할
- 각 행은 흰 배경 리스트 행 + 우측 화살표
- 상단은 검정 앱바, 닫기 액션 유지
- 각 항목 탭 시 상세 선택 화면으로 이동

상세 선택 화면:

- 상단은 검정 앱바 + 뒤로가기
- 항목 리스트는 흰 배경 행
- 현재 선택값은 노랑 체크 아이콘으로 표시
- 탭 즉시 저장 후 이전 화면으로 복귀

현재 설정 모델은 그대로 유지하고, 화면 구조만 예전 방식으로 바꾼다.

### 관심 비교

원형에 직접 대응되는 화면은 없지만, 시각 문법은 목록 카드의 확장판으로 맞춘다.

원칙:

- 앱바는 검정
- 배경은 노랑 또는 연회색이 아니라 목록과 같은 브랜드 바탕을 사용
- 비교 카드도 검정 외곽/흰 내부 인상 유지
- 가격, 거리, 가격 변화, 마지막 확인 시각의 위계를 목록보다 조금 더 비교 중심으로 정리

빈 상태도 현재의 단순 텍스트가 아니라 브랜드 톤의 안내 레이아웃으로 바꾼다.

## 디자인 시스템 변경

### 색상

`core:designsystem`은 더 이상 시스템 dynamic color를 기본 사용하지 않는다. 앱의 시각 정체성이 시스템 월페이퍼에 따라 흔들리면 안 된다.

필수 방향:

- `dynamicColor = false` 기본화
- 브랜드 컬러 명시적 확장
- 회색 단계 재정의
- Material color scheme에서도 브랜드 대비가 유지되게 on-color 세트 보강

확장 대상 색상:

- Primary yellow
- Black
- White
- Gray scale for dividers, labels, subtle backgrounds
- Stale/warning용 보조 노랑 또는 회색 tint

### 타이포

타이포는 기존 앱처럼 숫자 정보가 강하게 드러나야 한다.

필수 방향:

- 가격/거리용 큰 숫자 스타일 복원
- 카드 제목, 보조 라벨, 설정 행 텍스트의 위계 재정의
- 현재 기본 Typography의 평범한 Material 느낌 제거

### 컴포넌트 표면

- 기본 Card를 그대로 쓰기보다, 반복되는 브랜드형 카드 표면을 재사용 가능한 컴포저블 또는 스타일 함수로 정리
- 설정 행, 상세 선택 행, 상태 배너도 공통 스타일 토큰 사용

## 상호작용 설계

### 목록 화면

- 정렬 토글: 앱바 타이틀 탭 유지
- 카드 탭: 외부 지도 열기 유지
- 별 아이콘: 관심 토글 유지
- 관심 비교: 앱바 액션으로 유지
- 새로고침: 앱바 액션 유지

즉, 인터랙션 구조는 예전 단순함을 유지하되 현재 기능 버튼을 같은 문법으로 정렬한다.

### 설정 화면

- 목록 행 탭 -> 상세 선택 이동
- 상세 선택 탭 -> 즉시 저장 후 pop
- 닫기/뒤로가기 구조는 예전과 동일하게 유지

### 관심 비교

- 읽기 중심 화면으로 유지
- 카드 탭 동작은 당장 추가하지 않는다
- 이번 범위에서는 시각 복원과 정보 위계 정리에 집중한다

## 정보 위계

### 목록/비교 카드 공통

1. 가격
2. 거리
3. 주유소명
4. 브랜드/유종/가격 변화
5. 마지막 확인 시각

현재 화면은 텍스트가 모두 비슷한 무게로 보여서 목적형 앱 느낌이 약하다. 복원에서는 가격과 거리가 즉시 스캔되는 구조를 다시 만든다.

## 구현 범위

### 수정 대상

- `core/designsystem`
- `feature/station-list`
- `feature/settings`
- `feature/watchlist`
- 필요 시 `app/navigation` 수준의 경미한 진입 UX 조정

### 비범위

- 도메인 모델 변경
- repository 계약 변경
- stale 정책 변경
- watchlist 기능 의미 변경
- 지도 연동 로직 변경

## 검증 기준

### 시각 검증

- 세 화면 모두 원래 앱과 같은 브랜드 톤을 즉시 인식할 수 있어야 한다
- 목록 카드에서 가격과 거리가 가장 먼저 읽혀야 한다
- 설정 화면이 카드 섹션이 아니라 리스트 탐색 구조로 보여야 한다
- 관심 비교 화면이 새로 추가된 화면처럼 튀지 않고 기존 스타일 확장처럼 보여야 한다

### 기능 검증

- 기존 목록 기능이 유지되어야 한다
- 설정 저장 동작이 유지되어야 한다
- 관심 비교 진입과 표시가 유지되어야 한다
- 권한/GPS/stale/refresh 상태가 사라지지 않고 새 스타일에 맞게 표시되어야 한다

## 리스크와 대응

### 리스크 1. 원형 복원 과정에서 현재 기능 가시성이 떨어질 수 있음

대응:

- stale, refreshing, watchlist 상태를 독립 배너/보조 라벨로 체계화

### 리스크 2. Material 3 기본 컴포넌트에 너무 의존하면 원형 인상이 안 살아날 수 있음

대응:

- 브랜드 표면 컴포넌트와 색/타이포 토큰을 먼저 고정

### 리스크 3. 설정 2단계 복원 시 현재 단일 화면보다 이동이 늘어날 수 있음

대응:

- 원형 경험 복원이 목표이므로 구조는 되돌리되, 각 선택은 탭 즉시 저장 후 자동 복귀로 마찰을 최소화

## 결정

- 방향은 `거의 원본처럼 최대한 비슷하게`
- 세 화면 전체 복원
- 레이아웃과 인터랙션도 원형에 가깝게 복원
- 새 기능은 기존 문법에 맞춰 통합
