# Settings Detail Hierarchy Design

## Goal

`찾기 설정` 화면에서 `찾기 범위` 상세 화면으로 이동했을 때 시각 언어가 갑자기 바뀌지 않도록, 상세 화면을 별도 선택 페이지가 아니라 같은 설정 카드의 한 단계 깊은 확장판처럼 보이게 만든다.

## Problem

현재 상세 화면은 상단 설명 영역과 각 옵션이 모두 독립 카드처럼 보이기 쉽다. 그 결과 부모 화면의 `그룹 카드 + 플랫 로우` 구조와 분리되어 보이고, 내비게이션은 이어지지만 시각 구조는 다른 화면으로 점프한 인상을 준다.

## Chosen Direction

상세 화면은 새로운 조형을 도입하지 않는다. 부모인 `찾기 설정` 화면의 카드 문법을 그대로 유지하고, `탐색 설정` 그룹 안에서 `찾기 범위` 항목만 더 깊게 들어간 상태처럼 보이게 만든다.

핵심 원칙은 다음과 같다.

- 상세 화면의 중심 구조는 `카드 하나 안의 헤더 + 플랫 리스트`다.
- 옵션은 독립 카드가 아니라 같은 카드 안에서 구분선으로 나뉜 행이다.
- 선택 상태는 레이아웃 변경이 아니라 체크와 약한 포인트 컬러로만 표현한다.
- 부모 화면과 다른 미학을 추가하지 않는다.

## Layout

### Top Bar

- 기존 상세 화면과 동일하게 상단 바는 유지한다.
- 제목은 `찾기 범위`를 사용한다.
- 뒤로가기는 현재 탐색 구조를 유지한다.

### Main Card

- 본문에는 `LegacyChromeCard` 계열의 단일 카드 하나를 배치한다.
- 카드 상단에는 부모 그룹을 이어주는 `탐색 설정` 제목과 `주변 주유소를 불러올 반경을 정합니다.` 설명을 둔다.
- 카드 헤더 아래에는 얇은 구분선을 두고 옵션 리스트를 이어 붙인다.

### Option Rows

- `3km`, `4km`, `5km`는 카드 내부의 플랫 로우로 배치한다.
- 각 로우는 제목과 보조 설명을 갖는다.
- 로우 간 간격을 벌려 별도 카드처럼 보이게 하지 않는다.
- 카드 외곽선은 전체 그룹에만 존재하고, 각 로우는 내부 구분선으로만 나눈다.

## Interaction

- 각 옵션 로우는 전체 영역 클릭이 가능해야 한다.
- 선택된 항목에는 오른쪽 체크 아이콘을 노출한다.
- 선택 상태는 접근성 semantics에서 `Role.RadioButton`과 `selected=true`로 유지한다.
- 선택 상태 때문에 행 높이, 코너, 외곽선 구조가 크게 바뀌지 않도록 한다.

## Visual Rules

- 부모 화면의 노란 배경, 검은 외곽선, 흰 카드면, 굵은 제목 타이포를 그대로 유지한다.
- 옵션 행마다 `탐색 설정` 같은 overline을 반복하지 않는다.
- 선택 상태는 전체 배경색 전환이나 강한 하이라이트 카드가 아니라, 체크와 약한 노란 포인트로 충분히 표현한다.
- “설명 카드 1장 + 선택 카드 N장”처럼 보이는 구조는 금지한다.

## Component Strategy

- 카드 컨테이너는 기존 `LegacyChromeCard` 계열을 재사용한다.
- 옵션 리스트는 기존 `LegacyListRow`를 그대로 쓰지 않는다.
- 이유는 `LegacyListRow`가 자체 카드이기 때문에 카드 안 카드 구조가 다시 생기기 때문이다.
- 대신 `SettingsScreen`의 메뉴 로우 밀도와 유사한 전용 플랫 로우를 상세 화면 내부에 둔다.

## Error Handling And Edge Cases

- 옵션이 1개여도 동일한 카드 구조를 유지한다.
- 옵션이 늘어나도 여전히 하나의 카드 내부 리스트처럼 보여야 한다.
- 설명 텍스트가 2줄 이상으로 늘어나도 옵션 리스트와 시각 충돌이 없어야 한다.

## Testing

- 상세 화면 제목 `찾기 범위`는 상단 바에만 1회 노출되는지 확인한다.
- 카드 헤더는 `탐색 설정`과 설명으로 렌더링되는지 확인한다.
- 옵션 컨테이너는 단일 그룹 카드로 존재하는지 확인한다.
- 옵션 행은 수직으로 쌓이고 내부 선택 아이콘만 달라지는지 확인한다.
- 선택 상태가 `현재 선택` 같은 별도 문구 없이 체크 아이콘 중심으로 표현되는지 확인한다.

## Out Of Scope

- 바텀시트 전환
- 세그먼트 컨트롤 형태로의 변경
- 설정 정보 구조나 내비게이션 플로우 개편
