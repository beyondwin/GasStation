# 오프라인 전략

`data:station`은 주유소 검색 결과의 읽기 모델로 Room 저장소를 사용합니다. 저장 단위는 `StationCache`, `StationPriceHistory`, `WatchedStation` 세 가지입니다.

## 캐시 키

- 캐시 키는 `locationBucket + searchRadius + fuelType` 조합으로 구성합니다.
- `sortOrder`는 캐시된 결과 집합에 거리순 / 가격순 정렬을 적용할 수 있으므로 키에서 제외합니다.
- 브랜드 필터는 캐시 스냅샷을 읽은 뒤 클라이언트에서 적용합니다.
- `mapProvider`는 외부 앱 연동 대상만 바꾸고 가져오는 주유소 데이터셋은 바꾸지 않으므로 키에서 제외합니다.

## 신선도와 오래된 데이터 처리

- 신선도는 `StationCachePolicy`가 판정합니다.
- 오래된 데이터 기준 시간은 5분입니다.
- 캐시가 오래되었더라도 화면에는 계속 렌더링합니다.
- 새로고침이 실패하면 `DefaultStationRepository.refreshNearbyStations()`는 Room 스냅샷을 교체하지 않고 예외를 던집니다.
- 관찰 경로는 기존 스냅샷을 계속 읽기 때문에 마지막 성공 결과가 화면에 남고, 목록을 비우는 대신 UI는 오래된 상태를 유지합니다.

## 가격 히스토리와 관심 주유소

- `StationPriceHistory`는 주유소별 최근 성공 스냅샷을 보관해 가격 변화 배지를 계산합니다.
- 히스토리는 연료 타입까지 포함한 키 기준으로 주유소별 최신 10건만 유지합니다.
- `WatchedStation`은 사용자의 명시적 관심 목록을 보관하며, 현재 검색 결과에 없는 항목도 마지막 확인 정보와 함께 비교 화면에 남깁니다.
- watchlist 비교 화면은 최신 `StationCache`가 있으면 그 값을 우선 사용하고, 없으면 저장된 마지막 정보와 가격 히스토리만으로 fallback 읽기 모델을 만듭니다.

## 데모 flavor

- `demo`는 기본 검색 반경과 휘발유 유종에 맞는 결정적 주유소 스냅샷과 가격 히스토리를 미리 적재합니다.
- `demo`는 시드 캐시 버킷이 검토자 온보딩과 맞물리도록 고정된 서울 좌표도 함께 사용합니다.
- `demo`는 API 키가 필요 없습니다. 가짜 좌표와 시드 스냅샷/히스토리로 `prod`가 실제 네트워크 새로고침에서 사용하는 것과 같은 오래된 데이터 / 오프라인 의미 체계를 보여줍니다.
